  1. Que es un sistema de control de versiones?

    Es una herramienta en la cuál se organizan las diferentes versiones de un programa o configuración

  2. Cual es la diferencia entre git y github?

    Git es un sistema de control de versiones, mientras que GitHub es una plataforma de almacenamiento de código que usa Git como sistema de control de versiones

  3. Cómo creo una rama en git?

    Se utiliza el comando git branch [Nombre-Rama]

  4. Cómo clono un repositorio?

    Se puede clonar un repositorio con el comando git clone [URL]

  6. Cómo puedo generar un tag?

    Para poder generar un Tag, utilizaremos el comando git tag [nombre del tag]

  7. Cómo hago una mezcla de ramas?

    Se pueden fusionar gracias al comando merge

  8. Cómo puedo moverme a un commit especifico?

    Para moverse entre los commits, utilizaremos de vuelta el comando git checkout, es recomendable utilizar tags de los commits en este proceso.

  9. Cómo puedo borrar una rama?

    Para borrar ramas utilizamos el comando git branch -d [rama]

  10. Cómo puedo borrar un tag?

    Podemos borrar tags con el comando git tag -d [tag]

  12. Para que sirve el comando git remote -v?

    Este comando se utiliza para borrar archivos individuales o colecciones de archivos.

  13. Cómo le digo a git cual es mi email y correo?

    Con los comandos git config --global user.name (Nombre de usuario) y git config --global user.email (Correo electroníco)

  14. Cuales son los pasos para conectar git con github?

    Primero, deberemos crear una clave SSH, añadir esta clave a GitHub, y por último utilizar el comando ssh -T git@github.com

  15. Cómo puedo crear un alias en git?

    Se puede crear un alias utilizando en el comando git config --global una sección en el comando llamada alias.[nombre del alias]
